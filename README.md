# MTS Case: Объединение контуров зданий и восстановление высот

## О проекте
Проект объединяет два источника зданий (Source A и Source B) в единый геодатасет, очищает геометрию, сопоставляет дубликаты/пересечения, восстанавливает пропущенные высоты и формирует финальные слои для анализа радиопланирования.

Основной файл проекта: `код.ipynb`.

## Цель решения
- Получить единый набор зданий с корректной геометрией.
- Максимально заполнить поле высоты (`height_estimate`).
- Подготовить данные для аналитики по радиотеням и приоритизации зон модернизации сети.

## Логика пайплайна
1. Загрузка данных из Source A и Source B, базовая проверка строк/колонок.
2. Преобразование WKT/GeoJSON в геометрию, удаление служебных столбцов, проверка валидности.
3. Очистка данных (минимальная площадь, устранение шумовых объектов, контроль дублей).
4. Агломерация контуров Source A (объединение фрагментов одного здания через граф связности).
5. Пространственное сопоставление A и B (пересечения, fallback-правила для 1:1 и many-to-many).
6. Сборка финального набора (приоритет надежной геометрии, исключение избыточных дублей).
7. Восстановление высот:
   - прямой перенос из надежных источников, где это возможно;
   - ML-дозаполнение пропусков (без перезаписи надежных измерений);
   - отдельная обработка заглушек/аномалий.
8. QA-контроль: доля NaN/нулей, выбросы, распределения высот, источники заполнения.
9. Построение аналитических карт (радиотени, приоритеты зон, рекомендации для инженерной проверки).

## Входные данные
Ожидаются исходные CSV/геоданные для Source A и Source B (пути задаются в ноутбуке).

## Результаты
В зависимости от конфигурации ноутбука формируются:
- финальный геослой зданий (обычно `*.geojson`);
- таблица атрибутов (`*.csv`);
- интерактивные карты (`*.html`) для презентации.

## Запуск проекта

### Вариант 1: Google Colab
1. Откройте `самый_конечный.ipynb` в Colab.
2. Установите зависимости (ячейка `pip install ...`).
3. Подключите входные данные (Google Drive или локальная загрузка).
4. Запускайте ячейки по порядку сверху вниз.
5. Сохраните итоговые `geojson/csv/html` из финальных ячеек экспорта.

### Вариант 2: Локально (Jupyter)
1. Создайте виртуальное окружение:
   - Windows: `py -3 -m venv .venv`
   - macOS/Linux: `python3 -m venv .venv`
2. Активируйте окружение:
   - Windows: `.venv\\Scripts\\activate`
   - macOS/Linux: `source .venv/bin/activate`
3. Установите зависимости: `pip install -r requirements.txt`
4. Запустите Jupyter: `jupyter notebook`
5. Откройте `самый_конечный.ipynb` и выполните ячейки по порядку.

## Проверка корректности результата
Перед финальным экспортом рекомендуется проверить:
- количество объектов в финальном слое;
- число зданий с `height_estimate <= 0` и `NaN`;
- распределение высот (медиана, p95, максимум);
- долю высот, восстановленных ML.

## Примечания
- Для презентации лучше использовать облегченные интерактивные карты (grid/top-зоны), а не полный слой зданий.
- При публикации данных в веб важно учитывать размер GeoJSON (возможна предварительная оптимизация геометрии).

# MTS Case: Объединение зданий и восстановление высот

## О проекте
Проект объединяет два источника зданий (Source A и Source B) в единый геодатасет, очищает геометрию, сопоставляет контуры одного и того же здания и восстанавливает пропущенные высоты.

Основной ноутбук решения: `код.ipynb`.

## Цель
- Получить единый слой зданий с корректной геометрией.
- Заполнить поле высоты (`height_estimate`) без перезаписи надежных значений.
- Подготовить финальные слои для радиопланирования и презентационных карт.

## Логика решения (коротко)
1. Загрузка Source A и Source B, первичная валидация строк/колонок.
2. Гео-преобразование и очистка (служебные поля, невалидные/шумовые объекты, минимальная площадь).
3. Агломерация фрагментов зданий Source A (граф связности).
4. Пространственное сопоставление A↔B (strict/fallback, many-to-many).
5. Каноническая сборка финального слоя (B как база + новые A, где уместно).
6. Восстановление высот:
   - прямой перенос из надежных источников;
   - ML-дозаполнение пропусков;
   - обработка заглушек и QA-контроль.
7. Экспорт финальных результатов и построение карт (risk/priority/рекомендации).

## Входные данные
Используются исходные файлы Source A/Source B (CSV/Geo), пути задаются в ноутбуке `код.ipynb`.

## Выходные данные
В зависимости от конфигурации ноутбука формируются:
- финальный геослой зданий (`*.geojson`);
- атрибутивная таблица (`*.csv`);
- интерактивные карты (`*.html`).

## Запуск (Google Colab)
1. Откройте `код.ipynb` в Colab.
2. Установите зависимости (`pip install ...` в ячейке ноутбука).
3. Подключите входные данные (Drive или upload).
4. Выполните ячейки сверху вниз.
5. Сохраните итоговые `geojson/csv/html`.

## Запуск (локально, Jupyter)
1. Создайте виртуальное окружение:
   - Windows: `py -3 -m venv .venv`
   - macOS/Linux: `python3 -m venv .venv`
2. Активируйте окружение:
   - Windows: `.venv\\Scripts\\activate`
   - macOS/Linux: `source .venv/bin/activate`
3. Установите зависимости: `pip install -r requirements.txt`
4. Запустите Jupyter: `jupyter notebook`
5. Откройте `код.ipynb` и выполните пайплайн.

## Минимальный QA перед сдачей
- Общее число объектов в финальном слое.
- Число зданий с `height_estimate <= 0` и `NaN`.
- Ключевые квантили высоты (median, p95, max).
- Доля высот, восстановленных ML.

Список зависимостей проекта находится в `requirements.txt`.
